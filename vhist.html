yuxtapa version history & changelogs

Versions are listed in chronological order by release date, newest first.

If the version string in the title is underlined, it begins a new development branch (versions in different branches are never compatible).

If a version number is followed by a letter, eg. "version 4a", this is a fix/update in an old development branch (that of version 4 in the example). Any modifications and additions in version n[a-z] are always implicitly present also in the version listed right before that version (in the example's case, anything mentioned in version 4a is also present in version 5), but changes that are explicitly listed for a version of a new branch are never present in updates of any old branches (eg. things mentioned for version 5 do not appear in version 4b, even though 4b is released later than 5).

A change or a new feature should always imply the respective update in the manual. Please report any out-of-date information you find there, but note that the manual of the old branch (version 4x) is not being updated anymore.

A known issue is something that was noticed just prior to release (the aim is then to fix them by the next release), or something that appears to be a problem in some dependency rather than yuxtapa. Notifications of bugs are not added retrospectively to listings for old versions, though.

version 9 released Apr 19, 2012

Client:

  • Fixed connecting with IPv6, which has never worked since it hasn't been tested
  • New keycommand (default 'd') to show in the chat all the titles in the current view
  • Archer aimer path is cut off by obstacles and PCs
  • The flag map border is drawn more nicely (at least on terminals that support that!)
  • Consecutive identical messages are no longer repeated; also adjusted message scrolling speed
  • Fixed splitting non-ASCII unicode strings in the chat
  • If stored_chat_msgs is set to 0 the chat storage file now gets emptied
  • Client can now survive terminal resizes (you still want to avoid them, but at least you don't always have to restart)
  • known issue with urxvt from version 8 persists
  • known issue: (this has been around for ever) something strange is wrong with resetting the terminal colours on urxvt when the client closes; noticable when afterwards using vim with syntax highlighting

Server:

  • Additions:
    • New config entry 'ipv' that can be used to tell the server explicitly to use IPv4 or IPv6 (see the new server.conf template)
    • Poisoned PCs now have their view disturbed by sickly green flickering
    • Treasure item now has different colour when lying in water or in chasm
    • Players receive message about detecting traps (useful if the detected trap is covered by, e.g., the treasure item)
  • Changes:
    • Tweaked several game values (nothing radical)
    • Corpses now disappear if used by a scout to disguise
    • The accessibility check for team flags is applied to neutral flags, too
    • Improvements in dungeon-type map generation
    • In destroy mode, the boulders can no longer get generated around the purple team's initial flag (sorry if someone thought that was a cute touch)
    • If the teams have the same number of players, a newly joining player can now also be put into the purple team if the class they requested is not available in the green team
    • Drowning damage can be zero, too (meaning in practice that drowning death takes longer)
    • Traps are also automatically triggered if stumbled upon while blind
    • Team messages now indicate an empty team with '-' instead of printing nothing
    • Healers can randomly heal more than 1 hp per action
    • A player's death no longer causes watchers to be dropped
    • Player comparison (in shuffling and !howfair) no longer dependent on admin level
  • Fixes:
    • Yet another fix (and further complication) of the LOS algorithm
    • Fixed some small issues with dynamic lighting; there are issues still, but these won't be touched (see manual on known bugs)
    • Fixed how joining new players didn't receive the flag map right away
    • No more than one flag can now be generated per sector (bugfix)
    • Do not allow multiple shouts on the same turn (in fact: do not allow a shout if there is a sound at the PC's position already)
    • Torch symbol ',' for burnt out torch is now persistent
    • Fixed how dynamic lighting would not protect from known traps
    • Fixed splitting non-ASCII unicode strings in messages
    • Fixed a typo in message: "[...] her ownmagic missile"
    • Fixed how team deathmatch mode was not shown properly in !serverinfo
    • Fixed a bug in zap bouncing (situation where a direct bounce back hits a PC immediately)
    • Fixed (in a brutal way) a fatal bug in map generation
    • Fixed two issues with switching class (switching back to current class; and not being informed of new class if "killed" by intermission)
    • Let zombie bots be reaped

Bots:

  • Compile option to build IPv6 using bots
  • Bots can now swim
  • Bots now spawn as scouts and can use the disguise ability (not very smartly, but still)
  • Assassin bots can intentionally trigger light traps to blind enemies
  • Bots can light their torch off of a static torch
  • Wizard bots now relight their torch if it is extinguished by a water trap
  • Small but important improvements in bot walking:
    • Follow teammate scouts/miners when nothing else to do (does not apply to scouts and trappers)
    • Accept even worse terrain if really want to get to someplace
    • Do not follow very nearby sounds, follow voices always and all other sounds only randomly
    • Wait properly when crossing rough terrain or marsh
    • Chance not to walk included in random walking
    • Scouts won't rush towards far-off enemies

Compatability:

  • If you have some key binding mapped to 'd' in your client config, this will give trouble
  • Everything else should be compatible with v.8
  • new .conf templates

version 8 released Feb 5, 2012

Client:

  • Introduced the alternative "send mode"; see the new client.conf file
  • Message box now "dimmens" the older messages; implementing this has caused some changes in the server, too
  • Help screen shows space visibly (as 'SP') and tab properly (as '\t')
  • known issue: on a recent version of urxvt, the client is terribly slow to start (the cause has been narrowed down to the ncurses routine init_color; the problem does not arise on xterm or if the TERM variable is set so that colour values cannot be changed in urxvt)

Server:

  • Fixed a couple of logic errors in steal mode
  • Fixed issues with the LOS (and lighting) algorithm
  • Fixed background colour indicators, which had been broken for a few versions...
  • Bots are now identified as (B) in the team listings, instead of (G)
  • PCs dying in water don't leave a corpse
  • Bots trying to pick an unavailable class now get assigned another class (this is the way it has been documented to be all along, but apparently never was...)
  • Amount of drowning damage now depends on the max hps of the class (so classes with a lot of hps will, on the average, drown just as fast as others)
  • While drowning, PCs now create a splashing sound (regardless of class)
  • You can now hear (see) sounds when blinded
  • The mode of the next map now again uses the sector abbreviations

General:

  • Removed C++ boost from dependencies
  • Fixed a very minor, almost theoretical bug having to do with random numbers
  • Fixed some compilation issues with the latest gcc/glibc
  • Added some critical documentation, particularly on running multiple servers and using bots when the server uses a non-default port
  • Added a quite pointless but cute vim syntax file for yuxtapa maps
  • Added a script to generate empty map files for manual map design purposes

Bot improvements:

  • Suicide decision now based on time of isolation instead of interpreting a single view
  • Fixed the problem of bots spawning close to a chasm almost inevitably walking into it

Compatability:

  • client v.8 might experience some display issues when interacting with a server v.7

version 6 released Nov 17, 2011

Client:

  • previously typed message scrolling now works like in vim: only strings matching what you've already typed are suggested
  • default key for 'C'hat/'C'ommand changed into tab
  • typing mode can now be exited with escape, too, and the escape delay is set so low that this isn't annoyingly slow
  • message aliasing now replaces substrings, too!
  • client can be made to remember messages typed in previous sessions; new config variable stored_chat_msgs for this purpose

Server:

  • miners can destroy doors
  • made backstabbing requirements easier (a fix of a design flaw): remaining hiding is easier, and you can backstab blinded enemies
  • if you suicide while poisoned and have only one hp, the poisoner earns a kill ("scared to death" supersedes this, though)
  • messages now use long names for sectors, e.g. "south-east" instead of "SE"
  • in steal mode the location of the item is now updated if the item is dropped
  • bots are now informed of the turn duration setting (they used to just assume it's 250ms)
  • new game mode, team deathmatch
  • server stat printing modified a bit; in particular, accuracy is no longer 0% if the player never fired a single shot
  • new chat commands: !howfair, !lscmd
  • possibility of disabling specific commands in server config

Bot improvements:

  • many improvements on how bots move around
    • generally, bots will move towards enemies in sight
    • if there are no enemies, they will move towards any sounds they hear (which can be used to pull teammate bots to follow you; but note that there is no real pathfinding)
    • assassin and trapper bots will favour tiles which allow them to hide
    • if stuck (e.g. teleported into a tiny room), bots commit suicide
    • in lack of other controlling conditions, instead of moving entirely randomly, bots "remain on course" for the most part
  • special ability usage:
    • healer bots can heal themselves and bypassing teammates, and poison bypassing enemies
    • wizard bots cast mm when they deem it safe and useful
    • fighter bots use circle strikes in melee when no teammates are close
    • archer bots shoot arrows at enemies if they have a clear shot (can get hazardous)
    • miner bots can on occasion mine walls
    • trapper bots set traps every once in a while (and might sometimes accidentally disarm them, too)
    • mindcrafter bots might blink away from melee if they are low on hps
  • bots won't intentionally spawn as any of the "unruled" classes: scout, planewalker, combat mage (they can still spawn as these due to classlimiting)
  • you can check the bot executable version with "mrbrown -v"

compatability:

  • server and client are compatible with v.5 clients and servers, but bot compatability is broken!
  • new .conf templates with info on new features

version 3 released Nov 3, 2010

Client:

  • the "ouch feature" which allows automatic shout generation (with custom message) when the PC is damaged
  • in the same vein, "quick shouts": predefined messages that are sent upon hitting the keys F1—F12
  • players can now review their key mappings with the fixed key '?'. Consequently, no key can be remapped to '?'
  • repeated chat messages are now stored only once in the history
  • invalid original ctrl keybindings (^X) are now printed correctly in the error message
  • known issue: too long chat messages are not printed correctly

Server:

  • fixed the crash caused by a player joining a team and going back into spectating before spawning (have a feeling I've fixed this same issue at least three times...)
  • maps can now be stored&reloaded, and stored maps can be listed; new "mapdir" entry in server.conf
  • improvements to the "map generation testing" build: it is now minimal (without all the other server functionality even present in the binary) and can also store a map into a file
  • fixed a very stupid mistake due to which traps left by trappers were not linked to their planter
  • trappers can disarm traps
  • zaps now die off if they travel for too long without bouncing (no more lucky sniping from the other side of the map)
  • introduced undiggable walls
  • revamped house generation — from 550+ lines of code to 227 lines, yet the houses are now more varied!
  • when a player switches team or joins a team after being a spectator, their memory of the current map is cleared. Every non-spectator's map memory is cleared when the teams are shuffled
  • can determine map type (dungeon/outdoor) in config similarly as game mode; the value of this option also added to output of "!serverinfo"
  • a big revamp of the whole bot handling; end-user functionality is essentially the same but many delicate problems were fixed
  • fixed a stupid mistake that allowed giving too high player-IDs (ultimately making it impossible to connect to the server)
  • changed some wordings in some log messages about the config being wrong
  • known issue: in principle the !serverinfo replies can exceed the message length limit; this is not checked

other:

  • Mr. Brown updated (still doesn't do much of anything)
  • Katharsis! Got rid of auto*hell

compatability:

  • interaction version 1
  • first release to introduce map storing: map storage version 1
  • old Mr. Brown should not be used, replace with new one
  • all datafiles from v.2 are compatible
  • old configs work, but new templates for both client&server describe new features

version 2 released Oct 28, 2010

Client:

  • fixed the problem that the client, when awaiting connection, hogged 100% CPU
  • fixed parsing of IPv6 addresses (never tested!)
  • can now hit ^C to cancel connecting (if e.g. accidentally connecting to wrong server)
  • can now give server address directly as a command line argument
  • any message sent by the server as a response to a connection attempt was shown too briefly to be read; fixed this
  • fixed a display bug that made full-colour lit portals brown instead of blue
  • allowed remapping keybindings to ctrl+[key]
  • tweak of the getch() timeout value
  • dying now turns off walkmode
  • message typing recognises pageDown (empty line)
  • '-h' (and '--help') print some redundant info now
  • changed version compatability check (introducing "interaction version")

Server:

  • system to automatically compile a "server" to test map generation only
  • fixed a very stupid mistake that made all the maps seem stretched in the NW–SE direction
  • adjustment of the 'h' parameter and the thresholds in outdoor map generation (actual effects are many and difficult to explain)
  • an extensive revamp of the dungeon map generation
  • new multialgorithm for placement of houses
  • fixed the crashing caused by commands such as "!teambal " (with one space and nothing else after the command)
  • the command functionality to store/reload the current map ("!storemap X", "!nextmap X"); just the actual saving and loading is yet to be done...
  • Players who are kicked or banned now receive a message stating this
  • fireballs light carried torches (of any survivors who have a torch)
  • made swimming way more difficult (the way it was supposed to be, so actually this was a bug fix)
  • print an ASCII "logo" at startup, and a message about graceful shutdown (in stdout, not just in log)
  • don't print non-error log messages when running with "--pstats"
  • removed unnecessary '-lrt' from LDFLAGS (remnant from some past experimentation)
  • '-h' (and '--help') print some redundant info now
  • changed version compatability check (introducing "interaction version")

compatability:

  • interaction version 1
  • all data files from v.1 are compatible
  • v.1 configs work, but new client.conf template has added information on new features
