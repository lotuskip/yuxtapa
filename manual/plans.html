Planned Features

On this page are listed a number of ideas that might be implemented into some future release of yuxtapa. You are welcome to contribute new ideas or comment on these. Nothing major is going to happen, though.

Things that probably will appear in the next release, already:

  1. improvements with the map generation (more variety, more logic)
  2. trappers can also disarm traps
  3. adjustments, adjustments, adjustments — please send suggestions

Later plans that need further specification and perhaps more thinking:

Game features&rules related:

  1. more mining abilities (these make sense but might make miners too 'all-purpose'):
    • breaking windows just like walls (and even faster)
    • destroying closed doors completely by digging into them
    • destroying static torches similarly
    • clearing rough terrain
  2. to counter-balance that: make some walls undiggable (would look like regular walls but just not break)?
  3. fireballs light carried torches (a minor detail, but could be a nice touch?)
  4. acid traps (much like boobytrap, but doesn't vanish once triggered) (could call them something else, too; 'acid trap' comes from ADOM)
  5. illusory walls (walk-through but not see-through tiles)
  6. some kind of mind's eye type ability that would leave the PC standing vulnerable, but would allow moving the viewpoint quickly just like a spectator's (perhaps a new class for this?)
  7. new game modes; got suggestions?
Metagaming&interface related:
  1. Some behavioural client-end settings; for instance:
    • A setting to have an automatic shout generated if you are damaged (should be quickly toggleable on/off, and have an initial value defined in the config)i; note that the server prevents dead PCs from shouting, so single-hit kills would be silent
    • 'quick-shouts': some single key, the pressing of which generates a pre-defined shout, without the player having to type anything
  2. more statistics:
    • number of team kills
    • stat kills according to the method of killing (see ways to die)
    • suggestions?
  3. smarter, more advanced chat aliasing (suggestions?)
  4. possibility to remap the keys in a more advanced fashion (eg. to use Ctrl&Alt + key)
  5. use the space under the team info in the client screen for something significant; examples:
    • more in-game info, such as locations of flags belonging to own team
    • some info generated locally at the client end (like connection statistics)
    • some kind of longer message defined by the owner of the server (like "my server rules: ...")
    • instead expand the message box upwards (and the PC info etc., too, which then would perhaps need some more content)
    • or put the PC info etc. in the now-empty space and widen the message box?
  6. possibility to save the currently played map with a given name into a file and load a previously saved map to play (if the game happens to generate an awesome map, you wouldn't lose it; e.g. "!storemap coolmap", "!nextmap coolmap")
  7. an actual AI for the bots
  8. optional sound effects in the client end? (in addition to the visual '!'s you could actually hear a sound)
