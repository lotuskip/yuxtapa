Planned Features

On this page are listed a number of ideas that might be implemented into some future release of yuxtapa. You are welcome to contribute new ideas or comment on these. Nothing major is going to happen, though.

Game features&rules related:

  1. more mining abilities:
    • breaking windows just like walls (might even consider it a bug that you can't)
    • destroying closed doors by digging into them
    • turning floor into rough terrain (not always succesful, a chance of creating a chasm in dungeons, ...)
  2. acid traps (much like boobytrap, but doesn't vanish once triggered) (could call them something else, too; 'acid trap' comes from ADOM)
  3. arrows hitting traps could make them go off? (really relevant only with boobytraps and fireballs)
  4. illusory walls (walk-through but not see-through tiles)
  5. some kind of mind's eye type ability that would leave the PC standing vulnerable, but would allow moving the viewpoint quickly just like a spectator's (a new class for this)
  6. maybe take the facing variable into account more often (like divide attacks to frontal/flank/rear and adjust DV correspondingly)? This would be something you could turn on in the server config; players who dislike it could run 'old-school' servers where it's turned off.
  7. a new map type, 'indoor', which would be more architectural, less fractal/messy, but still randomly generated
  8. new game modes; got suggestions?
Metagaming&interface related:
  1. new server commands: !putteam and !setclass (for admins)
  2. more statistics:
    • number of team kills
    • record kills according to the method of killing (see ways to die)
    • "killing sprees" and notification of them (could be turned off in the server config)
    • suggestions?
  3. smarter, more advanced chat aliasing (suggestions?)
  4. use the space under the team info in the client screen for something significant; examples:
    • more in-game info, such as locations of flags belonging to own team
    • some info generated locally at the client end (like connection statistics)
    • some kind of longer message defined by the owner of the server (like "my server rules: ...")
    • instead expand the message box upwards (and the PC info etc., too, which then would perhaps need some more content)
    • or put the PC info etc. in the now-empty space and widen the message box?
  5. new map format that is directly editable with a text editor
  6. make the static entities (flags, torches, boulders, portals etc.) part of the stored maps (a nice idea but appears to demand a lot of rethinking and work — this was never planned!)
  7. an actual AI for the bots
  8. optional sound effects in the client end? (in addition to the visual '!'s you could actually hear a sound)
