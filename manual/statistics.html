Statistics

The server keeps various statistics on the performance of all the players. Normally these are 'hidden' in the servers memory, not visible to the players themselves, but they can be published by the server administrator. If you start the server with the extra command line argument '--pstats', the server will print to stdout all player statistics and exit.

The stats are supposed to be human-readable to some extent, but in particular easily analysable/representable with self-made scripts or programs.

The statistics table

For each player, the following information is included:

  • most recent nick
  • time information:
    • last seen (date and time)
    • total time spent connected to the server (h:min:s), minus intermissions
    • percentage of that spent spectating
    • percentage of that spent playing as each class (if non-zero)
  • performance information:
    • kills, per each separate way of killing (if not zero), and grand total
    • team kills, grand total
    • deaths, grand total
    • total healing received in hitpoints (includes self-healing)
    • archery accuracy-%
    • combat magic (zap) accuracy-% (since zaps can hit several times their accuracy can exceed 100%)

Hitting teammates does not count as hitting in the accuracy considerations. With archery, it is considered a hit if your arrow stops at a square where there is an opponent, even if the actual result is a 'miss' after the to-hit/DV consideration.

About the times, note that if the percentages don't quite add up to 100%, this is due to two reasons:

  • (main reason:) time spent dead, aka 'in the limbo' counts as time spent on the server but not as any of the listed percentages
    • this way it is possible to compare also how much time players 'waste being dead'
  • rounding errors; all percentages are rounded down to integers, and all times are rounded down to full seconds
