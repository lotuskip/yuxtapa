Client screen

A typical view of a yuxtapa client after succesfully connecting to a server and joining a team (explanations below):

Things you see here:

  1. The View in which everything happens
  2. Chat box showing things people say and messages from the server
  3. Event list: more messages from the server; compared to those in the chat these are
    • more 'private'; they can be different for each player, whereas the chat is global
    • more 'in-game', usually describing events that took place
    • more 'dynamic', usually describing events that took place just now, and scrolling the messages away automatically
  4. PC info: details about your player character
  5. Objective & team info: the game mode, objective status, and numbers of players in the teams
  6. Timers: time left to next Intermission and to next spawn of your team
  7. Empty space: this is where you type when you say something, see Chatting and shouting
  8. Indicators; see below
  9. Area indicator; tells in which part of the map you currently are (very important in many game modes!)
  10. Server info: name of the server if any, IP address, and whether you were identified upon connection ("identified") or not ("new id")
  11. Flag map showing where the spawn flags are (the ones where PCs spawn at in brighter colour)

For more information on the contents of the view (1), see Vision and hearing, Map symbols, and maybe Time system.

Note that the chat box (2) will be as large as your terminal size allows, while all other parts of the screen are of fixed size. The chat messages can be scrolled with the +/- keys (or whatever they are mapped to).

The listing (4) is empty if you are only Spectating. The details listed are explained thoroughly in the Rule mechanics page, but roughly speaking they are:

  • Your class; see Class overview
  • HP: hitpoints, so-and-so many out of a maximum of such-and-such (when 0, you die)
  • DV and PV represent your evasion abilities and level of worn protection, respectively
  • The last line describes the accuracy and damage of your melee attack
  • If you are poisoned (see Healer), an indicator appears here (in the screenshot it is missing)

The indicators (8) have an animation (by default a 'rotating' bar) which updates whenever the client receives a view update from the server (see Time system). If it runs 'smoothly', this means your connection is good. If it stops, this means your connection to the server is down, or that the server is in intermission mode. If the animation jerks alot, you are connected but the connection is poor.

There are two more indicators located at (8), not visible in the screenshot. One is the walking indicator. It is a white letter 'W', right next to the rotating bar. See Movement for more information on this. The third indicator (would be left-most) is your torch status, see PC Torch.
