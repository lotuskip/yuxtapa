Recording sessions

This is not really a feature of yuxtapa, but worth mentioning still.

It is possible to record sessions of playing yuxtapa (on the client side). Obviously this can be achieved with any desktop capturing software, but ones that work in a terminal only produce much smaller files (the recordings are basically text, not video; they can be made really small if compressed).

The script way

For GNU/Linux users, the best choise is the 'script' command. Basic usage (see the man page, 'man script', for more information): 

script -t 2> game.timing -a game.session
records your sessions into the file 'game.session', with timing information in the file 'game.timing'. Two separate files are a necessity when using script; to share the recording just tar it together, e.g.: 
tar -czf gamerec.tar.gz game.timing game.session
The command 
scriptreplay game.timing game.session
replays the session.

Note that script records everything that goes on in your terminal, so you must first invoke script (first of the above commands) and then run the yuxtapa client, and after playing, quit script to end the recording (either type 'exit' or hit CTRL+d).

The ttyrec way

Using the external program ttyrec you can record sessions into files that can be viewn with programs that exist for Windows, too, and there are even online viewers.

It is assumed that you can manage installing ttyrec on your system. To record a session, instead of executing the client in the usual manner, type 

ttyrec -e yuxtapa_cl
This will record the entire session. To view the recording, run 'ttyplay [name of the recording file]'. Add the flag '-s2' to double the playback speed. During playback, you can hit '+' or '-' to increase or decrease the speed, or '1' to reset the normal speed.

The command 'ttytime [file]' can be used to tell the length of a recording in seconds.

The major downside of ttyrec is this: you are limited to a screen size of 80x24. yuxtapa is perfectly playable in such a small screen, but the chat space is only one line high...
