Getting and installing

If you just want to try the game out real quick, the necessary steps are:

  • Download the source code tarball and extract it
  • Compile the server and the client as explained below
  • Create the directory ~/.config/yuxtapa
  • Copy the configuration template files server.conf and client.conf from 'yuxtapa/tmplates/' there
  • Run the server (yuxtapa_sv), leave it running
  • In another terminal run the client (yuxtapa_cl); you're then playing solo on your own local server

More precise instructions follow.

Downloading

You have one or two options:

  • Download the yuxtapa source code (from github) to compile it manually
  • If you are an Arch Linux user, just search for yuxtapa in the AUR
Contributions such as RPMs are welcome, and will be made available (or linked) if received.

Requirements

Here is a list of what libraries you need in order to build yuxtapa. Refer to the documentation of your Linux distribution to find out how to install them.

  • NCurses (typically called ncurses or libncurses)
  • ZLib (zlib)
If your distribution makes the distinction, these need to be the development versions (eg. ncurses-devel, zlib-devel).

Please see the Terminals page for information about what the yuxtapa client requires from the terminal emulator.

Compiling the source

yuxtapa has a separate executable for the server and the client. You can compile either or both. The AUR package installs both by default. Users of AUR know what to do. The following is for everybody else.

To build either only the client or only the server, run (in the directory 'yuxtapa' that is created upon extracting the source package) either of 

make client
and 
make server
respectively. Just 'make' compiles both. The client is called 'yuxtapa_cl' and the server 'yuxtapa_sv'. Both will be placed in the 'yuxtapa' directory.

Grabbing anything else you need from the package

There is no install script or Makefile rule: to "install" the game into your system, you must do something like 

mv yuxtapa_cl yuxtapa_sv /usr/bin
as root. There are no other "system files" beyond these two executables.

The source tarball contains other stuff that might be of interest to users, as well. Such as this manual (AUR users will have it installed in /usr/share/doc/yuxtapa/), the bot client (see the file 'testes/README' as well as the manual page on bots), and the template configuration files (see below).

Configuring and running

In order to get playing, you must configure the client. After that is done, you can run the client program with 

yuxtapa_cl
if you installed the executable into your path, or with 
./yuxtapa_cl
in the directory containing the executable otherwise.

Some notes about setting up a server are in the server configuration page.

On other platforms

The yuxtapa client has been succesfully compiled on OSX (10.6), and with some work the server should compile, too. OSX is not actually supported, though, so if you want to compile yuxtapa on your OSX, be prepared to modify the Makefile and possibly even some source code.

Some points you might find useful on OSX:

  • you will need a gcc style compiler installed; consider using MacPorts
  • zlib and ncurses should be present by default; if not, make them available
  • add '-D_XOPEN_SOURCE_EXTENDED' to the Makefile CPPFLAGS (this appeared to be required to include wide character support in ncurses)
  • as mentioned, the client has been compiled succesfully; the server code might require minor modifications to compile
