Getting and installing

Downloading

You have one or two options:

  • Download the yuxtapa source code (link on the homepage) to compile it manually
  • If you are an Arch Linux user, just search for yuxtapa in the AUR
Contributions such as RPMs are welcome, and will be made available (or linked) if received.

Requirements

Here is a list of what libraries you need to build yuxtapa. Refer to the documentation of your Linux distribution to find out how to install them.

  • NCurses (typically called ncurses or libncurses)
  • C++ boost (boost)
  • ZLib (zlib)
You will also need a sane C++ compiling environment full with the GNU autotools.

Please see the Terminals page for information about what the yuxtapa client requires from the terminal emulator.

Compiling the source

yuxtapa has a separate executable for the server and the client. You can compile either or both. Furthermore, you can compile with debug either enabled or disabled. (Debug being enabled results in a larger binary and perhaps a tiny performance drop. Regular users probably don't need to enable it.) By default both the server and the client are built, and debugging is disabled.

If you use AUR, these defaults are followed (but perhaps you know how to change that). The following instructions are intended for people not using AUR:

First extract the source package. Then, in a terminal, in the created directory 'yuxtapa-XX', run the following commands: 

aclocal
automake --add-missing
autoconf
This should create a bunch of files and print only a few messages. Next, run the created configure script (one of the following, for instance): 
./configure                                   [default: client + server, no debug]
./configure --enable-debug                    [client + server, with debug]
./configure --disable-client                  [just the server, no debug]
./configure --enable-debug --disable-server   [just the client, with debug]
./configure --prefix=/home/user/bin           [install in home dir]
etc., you get the idea. Running './configure -h' prints more options. If that ran okay without complaints, you can continue with 
make
to compile and, optionally, 
make install
(as root) to install (system-wide). Installing is not necessary to play or to run a server.

Grabbing anything else you need from the package

The source tarball contains other stuff that might be of interest to users, as well. Such as this manual (AUR users will have it installed in /usr/share/doc/yuxtapa/), the bot client (see the file 'testes/README' as well as the manual page on bots), and the template configuration files (see below).

Configuring and running

Next, in order to get playing, you must configure the client. After that is done, you can run the client program with 

yuxtapa_cl
if you installed the game into your path, or with 
./yuxtapa_cl
in the directory containing the executable otherwise.

Some notes about setting up a server are in the server configuration page.
