Getting and installing

Downloading

You have one or two options:

  • Download the yuxtapa source code (link on the homepage) and compile it yourself
  • If you are an Arch Linux user, just search for yuxtapa in the AUR

Requirements

Here is a list of what libraries you need to build yuxtapa. Refer to the documentation of your Linux distribution to find out how to install them.

  • NCurses (typically called ncurses or libncurses)
  • C++ boost (boost)
  • ZLib (zlib)
You will also need a sane C++ compiling environment full with the GNU autotools.

Please see the Terminals page for information about what yuxtapa requires from the terminal emulator.

Compiling the source

yuxtapa has a separate executable for the server and the client. You can compile either or both. Furthermore, you can compile with debug either enabled or disabled. (Debug being enabled results in a larger binary and perhaps a tiny performance drop.) By default both the server and the client are built, and debugging is disabled.

If you use AUR, these defaults are followed (but perhaps you know enough to change that). The following instructions are intended for people not using AUR:

First extract the source package. Then, in a terminal, in the created directory 'yuxtapa-XX', where XX is a version number, run the autools: 

autoconf
automake
These shouldn't print anything if everything goes smoothly. Next, run configure (one of the following, for instance): 
./configure                                   [default: client + server, no debug]
./configure --enable-debug                    [client + server, with debug]
./configure --disable-client                  [just the server, no debug]
./configure --enable-debug --disable-server   [just the client, with debug]
etc., you get the idea. If that runs okay without complaints, you can continue with 
make
to compile and, optionally, 
make install
as root to install. Installing is not necessary to play or to run a server.

Configuring and running

Next, in order to get playing, you must configure the client. After that is done, you can run the client program with 

yuxtapa_cl
if you installed the game, or 
./yuxtapa_cl
in the directory containing the executable otherwise.
